## 脚本执行引擎

### 1. 执行流程

```
开始 → 检查无障碍服务 → 打开目标 APP → 执行初始化操作 → 
循环执行 → 检查退出条件 → 结束
```

### 2. 初始化操作 (init_controls)

在脚本主循环开始前执行的操作：

- 打开搜索页
- 输入搜索关键词
- 点击搜索结果
- 等待页面加载

### 3. 循环执行

#### 3.1 循环类型

| 类型 | 配置 | 说明 |
|------|------|------|
| 次数循环 | loop_count | 执行指定次数 |
| 时间循环 | loop_time | 执行指定时间 |

#### 3.2 退出条件

- 满足退出条件
- 达到循环次数
- 用户手动停止
- 发生错误

### 4. 条件分支

根据条件执行不同的操作：

```json
{
  "conditions": [
    {
      "condition_id": "is_ad_playing",
      "condition_type": "text_exists",
      "condition_value": "广告|跳过",
      "true_branch": ["skip_ad"],
      "false_branch": ["continue_play"]
    }
  ]
}
```

### 5. 容错机制

#### 5.1 重试机制

- 元素未找到时重试
- 操作失败时重试
- 可配置重试次数

#### 5.2 备用策略

主匹配失败时，按顺序尝试备用策略：

```json
{
  "fallback_strategies": [
    { "strategy_type": "text", "strategy_value": "搜索" },
    { "strategy_type": "position", "position": { "x_ratio": 0.9, "y_ratio": 0.05 } }
  ]
}
```

#### 5.3 弹窗处理

自动识别并关闭弹窗：

- 广告弹窗
- 权限请求
- 更新提示
- 评分请求

## 远程控制功能

### 1. 心跳机制

#### 1.1 心跳请求

- 间隔：30 秒
- 内容：设备 ID、用户信息、当前状态

#### 1.2 心跳响应

- 待执行命令
- 配置更新
- 状态同步

### 2. 命令类型

| 命令 | 说明 |
|------|------|
| execute_script | 执行指定脚本 |
| execute_all | 执行所有线上脚本 |
| stop_execution | 停止执行 |
| update_config | 更新配置 |
| upload_screenshot | 上传截屏 |

### 3. 命令流程

```
后台发送命令 → 存入数据库 → 
APP 心跳获取命令 → 执行命令 → 
上报执行结果 → 更新命令状态
```
